# Primary sources

This file tracks the sources used to shape the atlas. Source inclusion does not mean that every conclusion in this repository is endorsed by the source author. Atlas entries cite the specific records they rely on.

**Access date for this release:** 14 July 2026

## SRC-001: OWASP Top 10 for Agentic Applications for 2026

OWASP Gen AI Security Project. A peer-reviewed risk framework for autonomous and tool-using systems.

https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/

**Used for:** comparison with current industry risk categories and terminology.

## SRC-002: NIST AI RMF Generative Artificial Intelligence Profile

Autio, Schwartz, Dunietz, Jain, Stanley, Tabassi, Hall, and Roberts. NIST AI 600-1, published 26 July 2024 and updated by NIST in 2026.

https://doi.org/10.6028/NIST.AI.600-1

**Used for:** risk-management framing, lifecycle thinking, documentation, measurement, and governance context.

## SRC-003: MITRE ATLAS

MITRE's knowledge base for adversarial techniques involving machine-learning systems.

https://atlas.mitre.org/

**Used for:** comparison with adversary behavior, technique-oriented records, and defensive mappings.

## SRC-004: OpenAI Agents documentation

Official documentation covering tools, orchestration, guardrails, state, observability, evaluation, and safety guidance.

https://developers.openai.com/api/docs/guides/agents

https://developers.openai.com/api/docs/guides/safety-best-practices

**Used for:** current implementation concepts and vendor-documented safety practices.

## SRC-005: Model Context Protocol security best practices

Official security guidance for authorization, confused deputy risks, token passthrough, server-side request forgery, session hijacking, local server compromise, URL handling, sandboxing, and scope minimization.

https://modelcontextprotocol.io/docs/tutorials/security/security_best_practices

**Used for:** tool-server trust boundaries, authorization, local execution, scope, and network controls.

## SRC-006: AgentDojo

Debenedetti, Zhang, Balunović, Beurer-Kellner, Fischer, and Tramèr. *AgentDojo: A Dynamic Environment to Evaluate Prompt Injection Attacks and Defenses for LLM Agents.* arXiv:2406.13352, revised 24 November 2024.

https://arxiv.org/abs/2406.13352

**Used for:** realistic prompt-injection evaluation, tool use over untrusted data, security test design, and the distinction between task utility and attack resistance.

## SRC-007: OpenAgentSafety

Vijayvargiya, Soni, Zhou, Wang, Dziri, Neubig, and Sap. *OpenAgentSafety: A Comprehensive Framework for Evaluating Real-World AI Agent Safety.* arXiv:2507.06134, revised 16 February 2026.

https://arxiv.org/abs/2507.06134

**Used for:** evaluation across real tools, multi-turn settings, multi-user tasks, and explicit unsafe-behavior measurement.

## Source policy

Primary papers, official specifications, standards bodies, maintainers, and verified disclosure records are preferred. News coverage and secondary summaries can help discover a topic but should not be the sole basis for an active threat record.

Each source should be reviewed again when an entry moves from E1 to E2 or when a cited specification changes.
